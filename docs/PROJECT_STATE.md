# Текущее состояние проекта

## Статус

Проект является рабочим браузерным MVP Tower Defense с центральной механикой цветовых слияний.

Игровой цикл:

1. игрок размещает чистые башни;
2. заранее видит состав и угрозу следующей волны;
3. объединяет совместимые цвета;
4. получает гибрид с новым названием, характеристиками и способностями;
5. защищает базу на протяжении 10 волн;
6. побеждает или проигрывает в зависимости от построения и расходования энергии.

## Технологии

- React 19;
- TypeScript 5.9;
- Vite 8;
- ESLint 9;
- CSS без отдельной UI-библиотеки;
- GitHub Actions для автоматических проверок.

## Архитектура

- `components/` — поле, панели, атлас и визуальные эффекты;
- `game/` — чистая логика боя, слияний, волн и баланса;
- `hooks/` — игровой таймер;
- `config/` — параметры башен, врагов и поля;
- `types/` — общие TypeScript-типы.

## Реализовано

- три чистых цвета;
- 27 двухцветных результатов;
- редкости 5/5–9/1;
- точные характеристики и способности в атласе;
- предварительная цена слияния;
- визуальная индикация доступности энергии;
- четыре архетипа врагов;
- 10 детерминированных волн;
- предварительный просмотр угрозы;
- минимальные CSS-снаряды, лазеры, ауры и подписи способностей;
- победа, поражение, продажа и перезапуск.

## Подтверждённые проверки

```bash
npm run build
npm run lint
```

Обе команды должны завершаться без ошибок.

## Целевой баланс

Ориентир — около 70% побед и 30% поражений после освоения слияния. Это гипотеза баланса, которую нужно подтверждать серией реальных прохождений.

## Ограничения текущей версии

- нет трёхцветных башен;
- нет сохранения коллекции и статистики партий;
- нет автоматической телеметрии побед и поражений;
- баланс ещё не подтверждён достаточным количеством прохождений;
- визуальные эффекты намеренно минимальны;
- нет финального звука и графических ассетов;
- пока не подключены 15 городов, благословения и валюта «ВЕРА».

## Важное правило разработки

Каждый новый этап выполняется в отдельной ветке и объединяется с `main` только после успешных автоматических и ручных проверок.
